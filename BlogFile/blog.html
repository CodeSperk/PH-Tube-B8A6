Scope of var, let & const

All of these three keywords are used to declare variable in Javascript. The scope or variables declared with var, let & const are varies. Scope can affect the accessibility from different parts of your code. Now, I will explain scope of each of these variable briefly. Here you go...

1. var: Var is Function Scoped.

If Var is declared in any function it can only be accessible within the function. But if you declare 'var' outside the function, it became a global variable, meaning you will be able to access it from anywhere in the code.
Moreover, Declaration of var is moved to the top of its container. It is called Hoisting.

2. let: let is block-scoped.

Variable Declaration with 'let' are block-scoped. So if you declare with 'let' keyword, you can access it only from the that block. Here block means parts enclosed by curly braces.

Variable 'let' are not hoisted to the top.

3. const: Const is also block-scoped.

It can neither be hoisted nor accessible from outside the block like 'let'

However, Const is Constant. The only difference between let and const is const can't be reassigned but let.

Cases of null and undefined

In Javascript 'null' and 'undefined' are two primitive data types.Having some similarities, there are some difference among them.Now I will explain them in brief. Here you go...

1. null: Used to assign Intentional Absence of Value and Resetting Variables

It is used to set the intentional absence of any object value. It is often used to indicate that a variable or property should have no value or that a function should return no meaningful result. Moreover, You can assign null to a variable to clear its value.

2. undefined: Default Value, Functin Return Value & Object Properties.

It represents the absence of a value in a variable or property that has not been assigned a value. It is a default value for uninitialized variables. It is also returned by functions that do not explicitly return a value.

Finally, Null and undefined are same as value but not as types. For example, `null == undefined` is true but `null === undefined` is false.

What is REST API?

A REST API, short for Representational State Transfer Application Programming Interface, is a widely-used approach to building web services and APIs. It is based on the principles of REST, which is an architectural style for designing networked applications.

Resources: In a REST API, resources are the central abstractions. Each resource is represented by a unique URL (Uniform Resource Locator), such as /books, /users, or /products. These URLs serve as endpoints that clients can interact with..
HTTP Methods: REST APIs rely on standard HTTP methods, including:
  • GET:Used to retrieve data from a resource.
  • POST:Used to create new resources.
  • PUT:Used to update existing resources.
  • DELETE:Used to remove resources.
Statelessness: REST APIs are stateless, meaning that every client request must include all the information required for the server to comprehend and handle it. There is no persistent session state stored on the server between requests, simplifying scalability and maintenance.
Uniform Interface: One of the key principles of REST is a uniform and consistent interface. This means that the same set of standard HTTP methods and status codes is used across different APIs, making it easier for developers to work with various RESTful services.
Representations: Resources in a REST API can have multiple representations, such as JSON, XML, or HTML. Clients can request the representation they prefer by specifying the appropriate "Content-Type" header in their requests.
Status Codes: HTTP status codes are used to indicate the outcome of a request. For example, a 200 OK status code indicates a successful response, while a 404 Not Found status code indicates that the requested resource was not found.
